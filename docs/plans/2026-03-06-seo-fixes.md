# SEO Fixes Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Corriger 3 catégories de problèmes SEO remontés par Google Search Console : 404 sur anciennes URLs, erreurs 5xx sur pages catégories/places, et pages dupliquées sans canonical.

**Tech Stack:** Next.js 15 App Router, TypeScript, Prisma, pnpm

### Task 7: Canonical sur la pagination des catégories

**Files:**
- Modify: `app/(front)/categories/[slug]/page.tsx`

**Step 1: Modifier la fonction `generateMetadata` pour ajouter le canonical**

Dans la fonction `generateMetadata` existante, ajouter `alternates.canonical` à l'objet retourné :

```typescript
export async function generateMetadata(props: CategoryPageProps): Promise<Metadata> {
  const params = await props.params;
  const searchParams = await props.searchParams;
  const page = parseInt(searchParams.page || "1", 10);

  const category = await prisma.placeCategory.findFirst({
    where: { slug: params.slug, isActive: true },
    select: { name: true, description: true, slug: true },
  });

  if (!category) {
    return {
      title: "Catégorie introuvable",
      robots: { index: false, follow: false },
    };
  }

  const title = `${category.name} - Toutes les places`;
  const description =
    category.description ||
    `Découvrez tous les établissements de la catégorie ${category.name}`;

  return {
    title,
    description,
    openGraph: {
      title,
      description,
      type: "website",
    },
    alternates: {
      canonical: `/categories/${category.slug}${page > 1 ? `?page=${page}` : ""}`,
    },
  };
}
```

Note : la signature de `generateMetadata` doit recevoir `searchParams` en plus de `params`. Vérifier si c'est déjà le cas.

### Task 8: Try/catch sur les pages catégories et places

**Files:**
- Modify: `app/(front)/categories/[slug]/page.tsx`
- Modify: `app/(front)/places/[slug]/page.tsx`

**Step 1: Wrapper la query catégorie dans un try/catch**

Dans `app/(front)/categories/[slug]/page.tsx`, la section de chargement de la catégorie (après les déclarations) :

```typescript
// Avant (actuel) :
const category = await prisma.placeCategory.findFirst({ ... });
if (!category) { notFound(); }

// Après :
let category;
try {
  category = await prisma.placeCategory.findFirst({ ... });
} catch {
  notFound();
}
if (!category) { notFound(); }
```

Faire de même pour le `Promise.all` qui charge les places :

```typescript
let places: typeof placesResult = [];
let totalPlaces = 0;
try {
  const [placesResult, total] = await Promise.all([...]);
  places = placesResult;
  totalPlaces = total;
} catch {
  // En cas d'erreur DB, afficher la page catégorie sans places
}
```

**Step 2: Même traitement dans places/[slug]/page.tsx**
